服务器集群监控系统设计
    为了能够及时监测到服务器的运行状态以及硬件健康状态，设计了服务器集群监控项目，该项目分为客户端和服
务器两个软件，客户端部署在需监控的服务器，远程监控服务器利用客户端传回数据进行分析，对于出现异常的
服务器能够及时发出警报。
    client 利用 shell 脚本（awk 等命令）来获取主机的基本信息，包括 cpu、mem、disk、proc、user，并按照约定的
数据格式对数据进行简单处理。
    利用 cJSON 获取配置参数，父进程向消息队列添加要读取的主机信息，子进程获取到消息队列的消息后添加对应
的任务到线程池中，任务函数通过调用 popen 函数执行脚本将获取到的主机信息添加到数据缓冲池中。
    client 利用 TCP 与 server 通信。当 server 离线时利用日志系统将数据缓冲池中的数据写到本地文件中，当 server
上线后，将日志文件中信息发送到 server 端。
    server 端采用 epoll+ 线程池模式设计。利用 cJSON 获取数据库等配置参数，将接收到的数据以及套接字传入任
务函数，利用线程池将数据写入数据库。利用哈希表存储 fd 与 IP，方便将数据插入到对应的表。
    日志系统利用 C++ 中 ostringstream 类实现。将继承自 ostringstream 类的自定义类构造函数封装成宏，生成临
时对象，在析构函数中将缓冲区中数据写入文件。
